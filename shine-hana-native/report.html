Migration Status

The migration of XSC design time artifacts to SAP HANA Native on SAP HANA Cloud is complete. Below are the detailed changes performed during the migration:

Key Migration Changes

Project Configuration Features:

  • Generated SAP HANA Native project structure
  • Migrated database artifacts to SAP HANA Native formats i.e hdbdd-> hdbtables, hdbviews, hdbindexes and hdbtabletypes
  • Converted design time artifacts to HDI-compatible formats
  • Generated MTA deployment descriptor

Data Type Conversion in Native Artifacts

  • ALPHANUM → NVARCHAR
  • TEXT → NCLOB
  • SHORTTEXT → NVARCHAR
  • CHAR → VARCHAR
  • BINTEXT → NCLOB

File extension changes from XS Classic

  • .hdbdd-> .hdbtable, .hdbview, .hdbindex, .hdbtabletype, .hdbdropcreatetable
  • .calculationview, .analyticview, .attributeview -> .hdbcalculationview
  • .hdbfunction, .hdbscalarfunction, .hdbtablefunction -> .hdbfunction
  • .analyticprivilege -> .hdbanalyticprivilege

Calculation Views with Parameters

  • Parameters without defined types default to String.

Series Entity

  • Series entity is not supported and is converted to a regular entity.

Data Type Conversion in Calculation Views

  • date() → daydate()
  • day() → daydate()
  • time() → daytime()
  • counter (Integer) → BIGINT

Synonym grantor service

  • Synonym grantor service will be added in mta descriptor if present in source project

Fuzzy Search Indexes

  • Enabled fuzzy search → Generates files under hdbindexes folder.
  • Implicit: Indexes from datatypes like SHORTTEXT and TEXT in SAP HANA 2.0 now require explicit handling in SAP HANA Cloud.
  • Explicit: If defined in config, explicit fuzzy index files are created.

Post Migration Steps

Migration of DB Artifacts from XSC to SAP HANA Native is complete. However, please review the following steps carefully before deployment:

  • SQL Syntax Changes in Procedures:
    • UPDATE FROM → MERGE INTO
    • TRUNCATE → DELETE FROM
  • CSV/hdbtabledata files
    • In the hdbtabledata files, remove namespace before csv files defined inside.
  • Unsupported Functions in Calculation Views:
    • CE_FUNCTION, CACHE, etc. are not supported.
    • Refer to SAP HANA Cloud Documentation for supported alternatives.
  • HdbSynonym, HdbSynonymconfig, and Hdbrole Files:
    • Verify target object parameters and names for accuracy before deployment.

Post Migration Steps - Service Layer

service.cds

  • Entity Imports: Confirm that all entities from DB are correctly imported.
  • Service Name: Each service.cds file uses the xsodata filename as the service name.
  • Duplicate Exposed Entities: If multiple services expose the same entity, review and remove duplicates unless explicitly needed.

service.js

  • Review for correct binding between entity operations and handler functions.

handlers

  • Confirm that all converted xsjs/xsjslib files are present.
  • Each file must export a function that binds to its relevant service events.
  • Verify functionality against original xsjs and xsjslib logic.
